SNOWFLAKE PIPELINE — SETUP GUIDE
=================================

1. FINDING YOUR ENV VARIABLES
------------------------------

SF_ACCOUNT
  Format : <orgname>-<account_name>   (e.g. myorg-myaccount)
  Where  : Snowflake UI → bottom-left corner (click your username)
           → "Account" shows the full identifier.
           Alternatively: Admin → Accounts → copy the "Account Locator" or
           the "Account URL" (strip https:// and .snowflakecomputing.com).
  Note   : Use the dot-notation form if your account is in a non-US region,
           e.g.  myorg-myaccount.eu-west-1

SF_USER
  Where  : Admin → Users & Roles → Users → your login name.

SF_PASSWORD
  Your Snowflake login password. For service accounts, consider using
  key-pair authentication (see Snowflake docs: Key Pair Authentication).

SF_DATABASE
  Where  : Data → Databases → pick the target database name.
  This is the database where the 4 DW tables (vehicles, suppliers,
  parts, quality_checks) and etl_run_log will be created / loaded.

SF_SCHEMA
  Where  : Data → Databases → <your database> → Schemas.
  Recommended: create a dedicated schema, e.g. PUBLIC or VEHICLE_DW.

SF_WAREHOUSE
  Where  : Admin → Warehouses → pick or create a warehouse.
  An X-Small warehouse is sufficient for this pipeline volume.

SF_ROLE
  Where  : Admin → Users & Roles → Roles.
  The role must have USAGE on the warehouse, database, and schema,
  plus CREATE TABLE and INSERT privileges on the schema.
  Leave blank in .env to use your user's default role.


2. REQUIRED PYTHON PACKAGES
-----------------------------
    pip install snowflake-sqlalchemy openpyxl


3. FILES IN THIS FOLDER
------------------------

  sf_connector.py       — Snowflake SQLAlchemy engine factory. Reads .env.

  get_source_data.py    — Queries source MySQL DB, writes source_data.xlsx
                          (one sheet per table). Run this before the pipeline.
                              python snowflake/get_source_data.py

  snowflake_pipeline.py — Core pipeline functions:
                            load_source_excel()      reads source_data.xlsx
                            run_source_assertions()  source-side validation
                            run_transform()          column mapping + derivation
                            push_to_snowflake()      INSERT to Snowflake tables
                            extract_from_snowflake() SELECT back for validation
                            run_post_assertions()    post-migration validation
                            push_to_run_log_sf()     writes to etl_run_log

  run_pipeline.py       — Orchestrator. Calls all steps in order and writes
                          one etl_run_log row per table after push.
                              python snowflake/run_pipeline.py

  verify_schema.py      — Pre-flight schema check. Compares actual Snowflake
                          column types and nullability against what the pipeline
                          expects. Run before the first pipeline execution.
                              python snowflake/verify_schema.py
                          Exits with code 0 (all pass) or 1 (mismatches found).

  source_data.xlsx      — Generated by get_source_data.py. Not committed to git.


4. TARGET TABLE DDL
--------------------
All 5 tables must exist in Snowflake before running the pipeline.
Translate the MySQL DDL (utils/test_data/sql_files/target_tables.sql) as follows:

  AUTO_INCREMENT          → AUTOINCREMENT  (or IDENTITY(1,1))
  TINYINT(1)              → NUMBER(1)       ← important: NOT BOOLEAN
  DATETIME / TIMESTAMP    → TIMESTAMP_NTZ
  DEFAULT CURRENT_TIMESTAMP → DEFAULT CURRENT_TIMESTAMP()
  ENUM('A','B')           → VARCHAR(n)      (Snowflake has no ENUM type)
  ENGINE=InnoDB           → remove
  FOREIGN KEY constraints → optional (Snowflake does not enforce them)
  Backtick quoting        → remove or use double-quotes

  Note on BOOLEAN vs NUMBER(1):
  If you create a column as BOOLEAN, pandas to_sql() will write Python
  integers (0/1) into it without error, but verify_schema.py will flag
  a TYPE MISMATCH. To fix an existing BOOLEAN column after creation:

      ALTER TABLE <table> ADD COLUMN <col>_tmp NUMBER(1);
      UPDATE <table> SET <col>_tmp = IFF(<col>, 1, 0);
      ALTER TABLE <table> DROP COLUMN <col>;
      ALTER TABLE <table> RENAME COLUMN <col>_tmp TO <col>;

  Affected columns: is_electric_vehicle, is_current, has_defect_flag,
                    has_defect, is_passed.


5. ETL RUN LOG
---------------
run_pipeline.py writes one row to etl_run_log per table after each push.
The following fields are populated:

  pipeline_name   — "snowflake_etl"
  table_name      — one of suppliers / vehicles / parts / quality_checks
  load_type       — FULL (if_exists="replace") or INCREMENTAL (watermark used)
  run_start       — UTC timestamp captured before push_to_snowflake()
  run_end         — UTC timestamp captured after push_to_snowflake()
  watermark_from  — passed through from run_pipeline(watermark_from=...)
  watermark_to    — passed through from run_pipeline(watermark_to=...)
  rows_extracted  — row count from source_data.xlsx before assertions
  rows_inserted   — rows actually written to Snowflake
  rows_failed     — rows dropped by source assertions (extracted - clean)
  rows_updated    — always 0 for full loads
  status          — SUCCESS or FAILED
  error_message   — populated only on exception, NULL on success

  The run log is written inside a try/finally block so a FAILED entry
  is always recorded even if push_to_snowflake() raises an exception.

  etl_run_log DDL for Snowflake:

      CREATE TABLE etl_run_log (
        run_id          NUMBER        NOT NULL AUTOINCREMENT PRIMARY KEY,
        run_start       TIMESTAMP_NTZ NOT NULL,
        run_end         TIMESTAMP_NTZ,
        pipeline_name   VARCHAR(100)  NOT NULL,
        table_name      VARCHAR(50)   NOT NULL,
        load_type       VARCHAR(15)   NOT NULL,
        watermark_from  TIMESTAMP_NTZ,
        watermark_to    TIMESTAMP_NTZ,
        rows_extracted  NUMBER        DEFAULT 0,
        rows_inserted   NUMBER        DEFAULT 0,
        rows_updated    NUMBER        DEFAULT 0,
        rows_failed     NUMBER        DEFAULT 0,
        status          VARCHAR(10)   NOT NULL,
        error_message   TEXT
      );


6. RECOMMENDED EXECUTION ORDER
--------------------------------
  1. python snowflake/get_source_data.py     # extract source → source_data.xlsx
  2. python snowflake/verify_schema.py       # confirm Snowflake schema is correct
  3. python snowflake/run_pipeline.py        # run full ETL pipeline
